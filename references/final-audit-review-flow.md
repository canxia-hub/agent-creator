# Agent Creator - 末尾审计复核流程（v1）

> 用途：在 Agent 完成通用创建、情景特化与深度设计后，进行最终审计复核。
> 目标：确认该 Agent 是否真正达到设计指标，并作为技能优化本身的复核时机。

---

## 一、定位

末尾审计复核流程是 Agent 创建的最后一道关口。

它有两个对象：
1. **被创建的 Agent**：是否达到设计指标
2. **agent-creator 技能本身**：流程是否完整、逻辑是否自洽、是否存在漏洞和矛盾

---

## 二、审计对象一：Agent 设计达标性

### A. 架构审计
- [ ] 六核心文件是否齐备
- [ ] 文件职责是否清晰
- [ ] 是否存在职责混杂

### B. 广度审计
- [ ] 是否已明确情景分类
- [ ] 是否已完成分支特化
- [ ] 是否已选择合理模板档位

### C. 深度审计
- [ ] 是否已建立目标理解机制
- [ ] 是否已建立行业方法论
- [ ] 是否已建立判断与优先级机制
- [ ] 是否已建立交付闭环
- [ ] 是否已建立经验沉淀机制

### D. 一致性审计
- [ ] 六核心文件之间无身份冲突
- [ ] 方法、风格、工具、记忆体系彼此兼容
- [ ] 用户对齐与安全边界一致

### E. 结果审计
- [ ] 该 Agent 能否清晰说明自己解决什么问题
- [ ] 该 Agent 能否说清自己的交付物是什么
- [ ] 该 Agent 能否说明何为完成、何为失败、何为下一步

---

## 三、审计对象二：技能流程本身的复核

### 1. 漏洞检查

检查当前技能是否缺少关键步骤，例如：
- 是否缺少从表层需求下潜到痛点的链路
- 是否缺少行业深度注入环节
- 是否缺少交付能力设计
- 是否缺少最终审计闭环

### 2. 矛盾检查

检查是否存在：
- 通用基准与情景特化冲突
- 广度层与深度层冲突
- 模板档位划分重叠或含糊
- 角色广度导引与六核心文件架构不兼容

### 3. 膨胀检查

检查是否存在：
- 文档太多但职责不清
- 同一概念多处重复定义
- 可合并但未合并的结构
- 会拖慢后续维护的重复层

### 4. 缺口检查

检查是否还缺：
- 场景模板母板的进一步细化
- 深度层在不同领域中的应用方式
- 最终标准与实际创建执行之间的桥接说明

---

## 四、最终判定等级

### 等级 A：成熟可用

- Agent 设计完整
- 技能流程闭环成立
- 无明显漏洞与矛盾

### 等级 B：可用但待补强

- 基本完成，但仍有局部漏洞或深度不足

### 等级 C：结构成立但未成熟

- 只有架构与广度，深度层不足

### 等级 D：半成品

- 流程缺环，Agent 或技能本身均未完成

---

## 五、产出要求

审计复核结束后，必须输出：

1. 结论等级
2. 已达标项
3. 漏洞项
4. 矛盾项
5. 下一轮修补建议
