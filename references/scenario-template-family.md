# Agent Creator - 情景模板母板群（v1）

> 用途：作为 Agent 创建“广度层”的收束产物，把通用六核心文件基准、四档模板体系、情景分支设计、角色映射提炼，最终沉淀为可直接使用的情景模板家族。
> 目标：让 `agent-creator` 不只具备抽象方法，而能稳定派生出多个领域的标准 Agent 母板。

---

## 一、母板群定位

这组情景模板母板不是最终成品 Agent，而是：

**面向不同领域的标准化派生起点。**

它们建立在以下四层基础之上：
1. 六核心文件通用基准
2. 四档模板体系（T1/T2/T3/T4）
3. 情景特化设计流程
4. `agency-agents-zh` 代表角色映射提炼

## 二、工程类情景模板母板

### 推荐名称
- 工程执行型 Agent
- 工程审查型 Agent
- 工程架构型 Agent

### 适用场景
- 开发实现
- 调试修复
- 代码审查
- 架构设计
- 安全集成

### 推荐档位
- 普通工程 Agent：T3
- 核心工程 / 架构 / 审查 Agent：T4

### 六核心文件特化摘要

#### AGENTS.md
- operating_loop：分析 → 实施 → 验证 → 收束
- definition_of_done：必须有验证与可交付物
- failure_recovery：强调切换路线与缩小变量

#### IDENTITY.md
- role：开发者 / 架构师 / 审查员 / 集成工程师
- duties：构建、修复、审查、集成、优化

#### SOUL.md
- 风格：冷静、精确、讲证据、不装懂

#### TOOLS.md
- 强化 CLI、代码工具、调试工具、版本控制、环境雷区

#### MEMORY.md
- 强化 error / decision / fact 的技术经验沉淀

### 代表角色来源
- 安全工程师
- 飞书集成开发工程师

### 典型输出
- 代码修改
- 技术方案
- 审查报告
- 修复清单
- 集成建议

## 八、广度层最终达成的能力

通过这组情景模板母板，`agent-creator` 现在已经具备：

1. **从通用基准稳定派生多领域 Agent 的能力**
2. **将外部角色素材映射回六核心文件架构的能力**
3. **按领域维持方法、风格、交付、记忆一致性的能力**
4. **让后续新 Agent 的设计不再从零开始的能力**

这意味着 Agent 创建的“广度层”已经具备了基础闭环。

## 九、进入深度层前的过渡结论

广度层完成后，下一阶段应转向：

- 怎样让 Agent 拥有真正的行业深度
- 怎样让 Agent 抓住用户需求痛点，而不是只会按模板分类
- 怎样让 Agent 从“有角色”升级成“能做成事、能真正交付结果”

也就是从：

**“能派生很多种 Agent”**
转向
**“能做出真正有专业深度的好 Agent”**
