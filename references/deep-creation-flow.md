# Agent Creator - 深度创建流程（v1）

> 用途：把深度层正式接入 `agent-creator` 主创建流程。
> 目标：让 Agent 创建不只完成架构与广度，还能经过深度设计，形成真正具备行业理解、用户痛点识别与交付能力的正式 Agent。

---

## 一、定位

本流程是在既有创建流程之上的 **深度增强层**。

它不替代：
- 六核心文件通用基准
- 四档模板体系
- 情景特化流程
- 情景模板分支设计

而是在这些步骤完成后，增加一条必须经过的深度设计链。

**结论**：
一个 Agent 只有在完成：
1. 通用基准
2. 情景特化
3. 深度设计

之后，才有资格被视为“成熟候选 Agent”。

---

## 二、深度创建流程总览

深度创建流程分为 5 步：

1. **深度前置诊断**
2. **用户痛点下潜**
3. **行业深度注入**
4. **交付能力设计**
5. **深度完成判定**

---

## 三、步骤详解

### 第 1 步：深度前置诊断

在进入深度设计前，必须先回答：

- 这个 Agent 是否只是“有角色”，但还没有深层工作结构？
- 它是否已经具备明确的情景分类？
- 它是否已经有了六核心文件的完整骨架？
- 它是否确实值得做深度设计，而不是停留在轻量档位？

#### 进入深度层的最低门槛

- [ ] 六核心文件已建立
- [ ] 已完成情景特化
- [ ] 已选择模板档位（通常是 T3 或 T4）
- [ ] 该 Agent 预期长期运行或承担重要任务

如果不满足以上条件，则不进入深度设计。

---

### 第 2 步：用户痛点下潜

调用 `user-painpoint-discovery-framework.md`，对该 Agent 所服务的任务类型进行深挖。

#### 必须完成的识别项

1. 用户通常以什么表层任务来提出需求？
2. 这些任务背后的真实目标是什么？
3. 用户最常见的约束是什么？
4. 用户最痛的点是什么？
5. 用户最在意的价值排序是什么？

#### 必须落到哪里

- `AGENTS.md`：在 operating_loop 中加入目标澄清 / 问题理解步骤
- `USER.md`：补充该类任务相关的长期价值偏好与风险偏好
- `MEMORY.md`：定义哪些痛点模式值得长期沉淀

---

### 第 3 步：行业深度注入

调用 `industry-depth-injection-model.md`，为该 Agent 注入行业深度。

#### 必须完成的注入项

1. 该行业的方法论是什么？
2. 该行业判断质量的标准是什么？
3. 该行业的常见失败模式是什么？
4. 如何通过经验沉淀机制替代部分 tacit knowledge？

#### 必须落到哪里

- `AGENTS.md`：methodology / operating_loop / failure_recovery / definition_of_done
- `IDENTITY.md`：duties / capability_panel / permissions
- `TOOLS.md`：tool_registry / local_traps / skill_pointers
- `MEMORY.md`：decision / error / fact 的行业化路由

---

### 第 4 步：交付能力设计

调用 `delivery-capable-agent-standard.md`，确保该 Agent 真正具备交付能力。

#### 必须回答的问题

1. 它的典型交付物是什么？
2. 它如何验证自己真的完成了任务？
3. 它如何在失败时收束任务？
4. 它如何给用户留下清晰下一步？

#### 必须落到哪里

- `AGENTS.md / definition_of_done`
- `AGENTS.md / task_specifics`（如适用）
- `MEMORY.md`（沉淀高质量交付路径）
- `USER.md`（记录用户偏好的结果形式）

---

### 第 5 步：深度完成判定

一个 Agent 在深度层完成后，必须能够回答：

- 它是否真正理解该类用户问题，而不是只会分类？
- 它是否拥有该行业的工作法，而不是只会行业术语？
- 它是否能交付结果，而不是只会提供解释？
- 它是否能通过经验沉淀逐步变强？

#### 深度完成清单

- [ ] 已建立目标理解机制
- [ ] 已建立问题建模与行业方法论机制
- [ ] 已建立优先级与判断结构
- [ ] 已建立交付闭环
- [ ] 已建立经验沉淀与复用机制

满足以上条件，才可视为“深度完成”。

---

## 四、与现有创建流程的衔接方式

### 推荐完整顺序

1. 建立六核心文件基准
2. 选择模板档位（T1/T2/T3/T4）
3. 完成情景特化
4. 完成情景分支映射
5. 进入深度创建流程
6. 通过深度完成判定
7. 再进入最终审计复核

### 原则

- 深度设计必须发生在情景特化之后
- 最终审计必须发生在深度设计之后
- 不允许跳过深度层直接宣称 Agent 已成熟

---

## 五、深度层与模板档位的关系

### T1 / T2

通常不强制进入完整深度流程。
仅适用于：
- 探索期 Agent
- 轻量辅助 Agent

### T3

建议进入深度流程，尤其是长期常驻 Agent。

### T4

必须进入深度流程。
主 Agent、管理型 Agent、核心协调型 Agent 必须具备深度层。

---

## 六、下一阶段接口

本流程完成后，下一阶段必须进入：
- 末尾审计复核流程
- 整体技能流程一致性核对
- 漏洞与矛盾排查
