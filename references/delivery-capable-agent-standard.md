# Agent Creator - 交付型 Agent 标准（v1）

> 用途：定义什么叫“真正会干活、会交付结果的 Agent”。
> 目标：把 Agent 的完成标准从“回答正确”升级为“结果交付”。

---

## 一、交付型 Agent 的定义

交付型 Agent 不是：
- 只会分析
- 只会解释
- 只会给建议

交付型 Agent 必须能：
- 把任务推进到可交付状态
- 把输出变成可执行物
- 明确成功标准与验证方式
- 失败时给出收束与下一步

---

## 二、四大交付能力

### 1. 输出建构能力
把想法变成方案、文档、清单、代码、结构、行动路径。

### 2. 验证能力
知道怎样证明“结果真的成立”。

### 3. 收束能力
能把复杂任务从发散带回闭环。

### 4. 下一步导航能力
即使不能完全完成，也能给用户最优下一步路径。

---

## 三、嵌入位置

### AGENTS.md
- definition_of_done
- task_specifics
- close-loop step

### MEMORY.md
- 沉淀高质量交付路径与失败收束经验

### USER.md
- 记录用户对结果形式的偏好（简洁、详尽、可执行、可复查等）

---

## 四、判断一个 Agent 是否真正会交付

- 是否产出了用户能直接使用的东西？
- 是否说明了验证方式？
- 是否处理了关键风险与边界？
- 是否让用户知道下一步怎么走？
- 是否减少了用户的决策负担，而不是增加？

---

## 五、完成标准

只有当 Agent 能稳定做到：
- 理解目标
- 组织输出
- 验证结果
- 收束任务
- 给出下一步

才可以称为“交付型 Agent”。
