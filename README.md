# Agent Creator - Agent 创建架构指南

本技能用于在通用使用场景下，**以 XML-first 的方式手动、审慎地创建 Agent**。

## 核心结论

完整的 Agent 创建，**必须包含六个核心文件**：

1. `AGENTS.md`
2. `IDENTITY.md`
3. `SOUL.md`
4. `USER.md`
5. `TOOLS.md`
6. `MEMORY.md`

这六个文件分别承载：
- **AGENTS**：运行契约、完成标准、安全边界、方法论、**通信纪律**
- **IDENTITY**：身份事实、职责、权限、关系
- **SOUL**：人格、语气、行为边界、模式切换
- **USER**：服务对象、身份验证、偏好、授权关系
- **TOOLS**：环境、工具地图、路径、服务约定
- **MEMORY**：长期记忆协议、经验沉淀、检索与写回机制

## 🆕 v3.1 新增：Agent 通信纪律

**所有新 Agent 从创建开始就具备正式通信能力。**

每个 Agent 的 `AGENTS.md` 必须包含 `<agent_communication_discipline>` 章节，内置：

| 模块 | 说明 |
|------|------|
| basic_rules | 6 条核心通信规则 |
| communication_flow | 消息头格式 + 处理流程 |
| silent_wait | DELEGATE 静默等待机制 |
| timeout_reference | 四种通信类型的超时参考 |
| silence_rules | 5 条静默规则（避免死循环） |

**通信类型**：
- **NOTIFY**：即发即弃
- **QUERY**：等待回复
- **DELEGATE**：静默等待，接收方主动回复
- **COLLABORATE**：多轮对话

**关联技能**：[agent-bridge](https://github.com/canxia-hub/agent-bridge)

## 创建原则

- **XML-first**：核心文件默认以 XML 为主格式
- **职责单一**：每个文件只承担自己的结构职责
- **手动审慎**：不依赖自动脚本，不跳过设计
- **泛化优先**：先设计完成标准、方法论、安全边界、记忆与用户传递机制
- **一致性校验**：六个文件之间必须彼此兼容
- **🆕 通信能力**：所有新 Agent 具备正式通信能力

## 推荐顺序

1. 明确 Agent 的长期职责与存在价值
2. 先定义完成标准、方法论、安全边界、记忆系统、用户传递机制
3. 创建六个核心文件骨架
4. 逐个设计六个核心文件
5. 做跨文件一致性校验
6. 再进入附属配置、渠道接入和扩展能力

## 一句话理解

Agent 不是一段 prompt，而是由**运行契约层、身份人格层、用户对齐层、环境能力层、长期连续性层**组成的结构系统。
